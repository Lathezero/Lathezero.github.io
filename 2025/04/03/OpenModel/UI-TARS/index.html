UI-TARS 本地部署

发表于2025-04-03|更新于2025-04-03|AgentUI-TARS
|总字数:517|阅读时长:2分钟|浏览量:

先决条件

  • Node.js >= 20
  • pnpm >= 9

技术栈

这是一个单存储库项目，包含以下技术：

  • 跨平台框架：Electron
  • 界面：
    • React
    • Vite
    • Chakra UI V2
  • 状态管理和通信：
    • Zustand
    • @ui-tars/electron-ipc
  • 自动化框架/工具：
    • nut.js
  • 测试框架
    • Vitest
    • Playwright

项目结构

注意：由于 Electron Forge 之前不支持 Pnpm 的提升机制（electron/forge#2633），要求将 src 目录放在顶级目录中，因此 src 目录位于顶级目录而不是 apps/{main,preload,renderer} 目录中。

安装依赖项

运行应用程序

应用程序启动后，您可以在应用程序内看到 UI-TARS 界面。

注意：在 MacOS 上，您需要授予运行命令所使用的应用（例如 iTerm2、Terminal）权限。

主进程重载

默认情况下，pnpm run dev 只有前端热模块替换（HMR）热更新。如果您需要在调试期间同时重载主进程，可以执行 pnpm run dev:w。

构建

在当前系统中运行 pnpm run build，它将输出到 out/* 目录中。

要构建其他系统的产品，请运行：

  • Mac x64：pnpm run publish:mac-x64
  • Mac ARM：pnpm run publish:mac-arm64
  • Windows x64：pnpm run publish:win32
  • Windows ARM：pnpm run publish:win32-arm64
